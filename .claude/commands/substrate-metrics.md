# Substrate Metrics

Review pipeline performance metrics, compare runs, and tag baselines.

## When to Use

Invoke this command when you need to:
- Review recent pipeline run performance (tokens, cost, timing, review cycles)
- Compare two runs to identify regressions or improvements
- Read detailed analysis reports generated by the supervisor
- Tag a run as the new baseline for future comparisons

## Viewing Recent Runs

```bash
npx substrate metrics --output-format json
```

Returns an array of recent run metrics records. Key fields per run:

| Field | Description |
|-------|-------------|
| `run_id` | Unique run identifier |
| `status` | `completed`, `running`, `failed` |
| `total_input_tokens` | Total input tokens consumed |
| `total_output_tokens` | Total output tokens produced |
| `total_cost_usd` | Total cost in USD |
| `wall_clock_seconds` | Total wall-clock time |
| `total_review_cycles` | Total review/fix cycles across all stories |
| `stories_succeeded` | Count of stories that passed |
| `stories_failed` | Count of failed stories |
| `is_baseline` | `1` if this run is tagged as baseline |

### Example: summarize the last 5 runs

```bash
npx substrate metrics --output-format json | jq '
  .[:5] | .[] | {
    run_id: .run_id[0:8],
    status: .status,
    cost: .total_cost_usd,
    tokens: (.total_input_tokens + .total_output_tokens),
    review_cycles: .total_review_cycles,
    succeeded: .stories_succeeded,
    failed: .stories_failed
  }
'
```

## Comparing Two Runs

```bash
npx substrate metrics --compare <run-id-a>,<run-id-b> --output-format json
```

Returns a comparison object with percentage deltas between the two runs:
- Negative delta = run B was better (fewer tokens, lower cost, etc.)
- Positive delta = run B was worse (regression)

### Example: compare against baseline

```bash
BASELINE=$(npx substrate metrics --output-format json | jq -r '.[] | select(.is_baseline == 1) | .run_id' | head -1)
LATEST=$(npx substrate metrics --output-format json | jq -r '.[0].run_id')

npx substrate metrics --compare "$BASELINE,$LATEST" --output-format json | jq '{
  tokens_delta_pct: .tokens_pct,
  cost_delta_pct: .cost_pct,
  review_cycles_delta_pct: .review_cycles_pct,
  wall_clock_delta_pct: .wall_clock_pct
}'
```

## Reading Analysis Reports

The supervisor generates analysis reports after each completed run. To read them:

```bash
npx substrate metrics --analysis <run-id> --output-format json
```

Returns the supervisor's analysis for the given run, including:
- `recommendations` â€” list of optimization recommendations
- Each recommendation has `type`, `phase`, `story_key`, `description`, and metric details

### Analysis report fields

| Field | Description |
|-------|-------------|
| `run_id` | Run this analysis covers |
| `recommendations` | Array of optimization findings |
| `recommendations[].type` | `token_regression`, `review_cycles`, `timing_bottleneck` |
| `recommendations[].phase` | Which pipeline phase had the issue |
| `recommendations[].story_key` | Story where the issue occurred |
| `recommendations[].description` | Human-readable explanation |

### Example: check if analysis is available and summarize recommendations

```bash
ANALYSIS=$(npx substrate metrics --analysis <run-id> --output-format json 2>/dev/null)
if [ $? -eq 0 ]; then
  echo "$ANALYSIS" | jq '{
    recommendations: (.recommendations | length),
    types: [.recommendations[].type] | unique
  }'
fi
```

## Tagging a Baseline

Mark a specific run as the baseline for future comparisons:

```bash
npx substrate metrics --tag-baseline <run-id>
```

After tagging, the run's `is_baseline` field is set to `1`. Only one run is the active baseline at a time. Future `--compare` calls without explicit run IDs will use the tagged baseline.

### Tagging strategy

Tag a baseline when:
- A run achieves the best performance you've seen
- After a significant prompt improvement is merged
- At the start of a new development sprint

```bash
# Tag the most recent completed run as baseline
LATEST=$(npx substrate metrics --output-format json | jq -r '[.[] | select(.status == "completed")][0].run_id')
npx substrate metrics --tag-baseline "$LATEST"
echo "Tagged $LATEST as new baseline"
```

## Experiment Reports

When the supervisor runs experiments (via `--experiment` flag), results are stored in:
```
_bmad-output/supervisor-reports/<run-id>-experiments.md
```

Read with:
```bash
cat _bmad-output/supervisor-reports/<run-id>-experiments.md
```
