# Story 25-4: Contract Declaration in Story Creation

Status: review

## User Story

As a pipeline operator,
I want stories to declare their interface contracts (exports and imports) during creation,
so that the pipeline can build a dependency graph and detect cross-story contract mismatches before they cause runtime integration failures.

## Background

When parallel stories share a message contract (queue, API, schema), neither story knows about the other's interface design. This caused the P0 cross-module schema mismatch in the v0.2.29 run: judge-agent published a different JudgeResult shape than check-publisher expected, and the pipeline couldn't detect it because build passed and review was per-story.

This story adds contract awareness at story creation time. The create-story prompt instructs agents to identify what schemas/interfaces the story exports (creates) and imports (consumes from other stories). These declarations are stored in the story file and persisted in the decision store for downstream use by the dispatch ordering engine (Story 25-5) and verification gate (Story 25-6).

## Acceptance Criteria

### AC1: Create-Story Prompt Enhancement
**Given** the create-story prompt template at `packs/bmad/prompts/create-story.md`
**When** the story agent creates a new story
**Then** the prompt instructs the agent to identify `exports` (schemas/interfaces the story creates) and `imports` (schemas/interfaces the story consumes from other stories)

### AC2: Story File Interface Contracts Section
**Given** a story file generated by the create-story agent
**When** the story has cross-story dependencies
**Then** the story file includes an `## Interface Contracts` section listing exports and imports with schema names and file path locations

### AC3: Contract Declarations in Decision Store
**Given** a story file with an Interface Contracts section has been created
**When** the pipeline processes the create-story result
**Then** the pipeline parses the Interface Contracts section and stores contract declarations in the decision store with category `interface-contract`

### AC4: Contract Declaration Schema
**Given** a contract declaration stored in the decision store
**When** the declaration is queried
**Then** it includes: contract name, direction (export/import), schema location (file path), story key, and optionally message transport (queue name, API path, etc.)

## Dev Notes

- The create-story prompt is at `packs/bmad/prompts/create-story.md` — add an "Interface Contracts" section to the prompt instructions
- The story template can be extended with `## Interface Contracts` — but keep it optional (stories with no cross-story deps omit it)
- After create-story returns, parse the story file for the Interface Contracts section in `orchestrator-impl.ts` (in the `processStory` flow, right after create-story completes)
- Use a simple markdown parser: look for `## Interface Contracts` heading, then parse bullet items with format like:
  - `- **Export**: SchemaName @ src/path/to/file.ts (queue: some-queue)`
  - `- **Import**: SchemaName @ src/path/to/file.ts (from story 25-X)`
- Store each declaration as a decision with category `interface-contract`, key `{storyKey}:{contractName}`, value as JSON with { direction, schemaName, filePath, storyKey, transport? }
- The compiled create-story workflow is at `src/modules/compiled-workflows/create-story.ts` — the result parsing happens there
