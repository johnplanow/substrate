# Acceptance Gate — adversarial red-team (A5.1, 2026-07-08)

*Independent adversarial subagent (separate lineage from the implementer) attacked the
shipped code (v0.21.9) + the 12-item evasion catalog. 9 findings; dispositions below.
The accounting spine and injection-safe render argv were found genuinely solid; the
exploitable surface concentrates where the design's own not-yet-built defenses live.*

## REFUTED (defenses confirmed solid)
- Render-command placeholder injection — argv-split-before-substitute, no shell (`contract.ts`). Literal tokens.
- Judge context leakage — params expose only the trusted journey + artifacts + telemetry storyKey; no diff/files_modified/story text/transcript can reach the prompt.
- Tag omission — `unclaimed`/`unwalked` escalate at epic close (where the invariant enforces; F2/F3 were the enforcement holes, now fixed).
- Spec tamper in worktree+merge mode — AcceptanceSpecCheck catches divergence either direction pre-merge.

## Deferred as accepted-risk / follow-up rows
- **A5.4** (new): full snapshot-compare for `--no-worktree` blocking mode (F3 complete close). Design-doc-first.
- **A6 canaries** (already planned): the structural close for F1 — revert a real wiring commit, re-run, verdict must flip. **A7 blocking-flip is gated on this.**
- **Phase-4 interactive driver** (already out-of-scope): the only real close for "follow-affordance-end-to-end" end-states (F1 sub-case) — a static render cannot prove end-to-end wiring.
