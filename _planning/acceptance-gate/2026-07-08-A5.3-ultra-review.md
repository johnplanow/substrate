## Findings: 1 (severity: normal) — RESOLVED

### U1 — F1 render-target tripwire is inert in the happy path AND crashes on `web` surfaces
`src/modules/implementation-orchestrator/orchestrator-impl.ts` (the Ship-9 F1
tripwire in `runAcceptanceStage`). Two independent defects in code shipped this
session:

- **U1a (inert):** the tripwire read `checkGitDiffFiles(worktreeDir)`, which
  sees only uncommitted/untracked changes. But `runAcceptanceStage` runs on the
  finalization path, AFTER commit-first (H0.1) has already committed the dev
  agent's output — so all git queries return empty, `gamedTargets` is always
  `[]`, and the warning never fires. The exact evasion it targets (a story
  rewriting its render script to hardcode passing output) is precisely the
  change commit-first commits before the check runs. The tripwire was dead.

- **U1b (crash):** the tripwire loop iterated `journey.surfaces` without
  skipping `web` (the render loop 20 lines below does). A journey declaring
  `web` + a contract with a `web:` block (both schema-legal, both in the design
  brief) makes `def` a `WebSurface` (`serve`/`ready`, no `render`), so
  `def.render.trim()` throws `TypeError`. The throw aborts the whole stage for
  that story; the upstream best-effort catch swallows it and sets
  `acceptanceOutcomes = []` — silently dropping EVERY journey for the story,
  surfacing in blocking mode as a misleading `journey-unwalked`.

**Resolution (Ship 11, v0.21.12):**
- U1a: `runAcceptanceStage` now takes the COMMITTED diff as a parameter; the
  call site passes `recaptureChangedFiles(...)` (baseline..HEAD ∪ dirty — the
  same ground truth the H7 disclosure gate uses). The reviewer's suggested
  in-place `recaptureChangedFiles` call was adjusted: that helper lives in
  `processStory`'s scope, not the top-level `runAcceptanceStage`, so the diff
  is threaded in as an argument instead.
- U1b: `if (surface === 'web') continue` added at the top of the tripwire loop
  (mirrors the render loop), plus a `def.render !== undefined` guard as
  belt-and-braces.
- Regression tests (integration harness): a `web`+`cli` journey now judges its
  cli surface walked-pass without crashing (would have thrown pre-fix); the
  tripwire warns on a committed render-target edit (would have been silent
  pre-fix). Full suite 583/11,436, matrix 23/23, eval 100%.

## Assessment

Ultra found what A5.1 and A7 could not: A5.1 read the code statically and
accepted F1 as a "partial" mitigation without noticing it was inert against

didn't touch a render target, so neither defect fired. This is exactly the

sharp reminder that "shipped with tests + a passing gate" is not "correct"

verdict×tier policy drew no issues.
